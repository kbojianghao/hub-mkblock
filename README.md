# hub-mkblock

## 项目简介

`hub-mkblock` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为一组静态页面的集中存放与分发场所。

这些页面可能包含信息展示、工具页面、说明文档等，各自独立，彼此无关联。仓库的主要目的是提供一个简洁、可访问的托管环境，便于快速访问和引用这些页面。

---

## 目录结构

```
hub-mkblock/
├── pages/           # 存放所有独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
├── assets/          # 页面所需的资源文件（如图片、样式、脚本）
│   └── ...
└── README.md        # 本文件
```

- `pages/`：每个 HTML 文件对应一个独立的页面，文件名通常反映其内容或用途。
- `assets/`：页面引用的公共资源，按需组织，不强制特定结构。

## 页面归档说明

- 本仓库中的页面均为静态 HTML，不依赖后端服务。
- 页面之间无相互引用或依赖关系，可单独访问。
- 归档目的仅为保存与分享，不提供任何形式的更新保证或功能承诺。
- 部分页面可能包含外部资源（如 CDN 库），访问时需保持网络连接。

## 使用方式

您可以通过以下方式访问仓库中的页面：

1. **直接浏览**：在浏览器中打开 `pages/` 目录下的 HTML 文件（需通过 GitHub Pages 或本地服务器）。
2. **克隆仓库**：`git clone https://github.com/your-username/hub-mkblock.git`，然后在本地打开对应文件。
3. **部署至其他静态托管服务**：将 `pages/` 目录整体上传至您选择的静态站点托管平台。
